多群联动管理Bot — 版本历史
============================

V1.0.6-beta.1 (2026-06-28)
--------------------
▸ 框架重构：core/commands.py（~1340 行）拆分为 core/dispatcher.py + 6 功能模块
▸ 注册制架构：dispatcher 提供 register_command() / register_event() 接口，功能模块无需修改框架代码
▸ 统一 Bot API：语义化封装 send_message() / kick() / ban() / unban() / send_image() 等协议操作
▸ 功能模块化：
  - features/basic/（基础功能）：help.py、config_cmd.py、admin.py
  - features/punish/（处罚系统）：punish.py（含黑名单入群监听）、rp.py、history.py
  - features/verify/（进群验证 — 新增）
  - features/render.py：Pillow 图片渲染（从 core/ 迁入）
▸ 进群验证系统（新增，~2100 行）：
  - 答题验证（verification）：新成员入群后多模式答题，支持计算题 / 问答题，Block 机制支持 ALL / RANDOM / SELECT 出题策略
  - 加群审批（approval）：正则匹配入群申请 comment，自动同意 / 拒绝 / 忽略
  - 图片卡片渲染：题目卡片 + 验证指引图（Pillow），独立于文本渲染管线
  - 每配置独立验证方案 + 每群独立开关，非 Bot 审批入群支持独立超时
  - 指令：/verify（on/off/status/welcome/maxerr/retry/timeout/bottimeout/block/question 子命令）
  - 指令：/approval（on/off/status/regex/reject/mismatch/welcome 子命令）
  - 监听 request.group_add 审批加群请求、notice.group_increase 触发答题、message.group 捕获答案
▸ /config list 指令：列出已有配置，支持图片渲染（Pillow 表格）
▸ 消息发送改用数组格式：符合 OneBot v11 message 段规范
▸ 命令显示优化：调整 help / admin 输出格式，提升可读性
▸ 修复：新注册命令在 command.json 缺失时无法解析
▸ 修复：help 命令配置分组显示异常
▸ 修复：render / verification 模块小问题

V1.0.5 (2026-06-25)
--------------------
▸ 多配置架构：单管理组 → 多配置独立存储（data/<配置名>/punish/）
▸ 命令配置系统：data/command.json 全局 + 各配置 command.json 覆盖，支持别名
▸ 权限模型扩展：三级（0/1/-1）→ 多级（0/≥1/-1），各命令支持 min_level
▸ 指令变更：/group → /config（new/rename/notify/set/remove/group 子命令）
▸ help 重构：卡片化分组（圆角背景），添加 AGPL v3 声明
▸ 新增旧版数据迁移工具 tools/migrate.py
▸ 代码分层：将指令处理拆分为 render.py / commands.py

V1.0.4 (2026-06-24)
--------------------
▸ 初始发布
▸ 基于 OneBot v11 协议的全新 Python 实现，支持在 Linux / Windows / macOS 任意平台运行
▸ 通过反向 WebSocket 与 OneBot 实现端通信，异步处理事件与指令
▸ 支持踢人、禁言、警告、撤销、记录查询等完整处罚操作
▸ 支持黑名单自动拦截入群
▸ 与 QFun Java 版指令集、数据格式、权限模型完全兼容，数据文件可互换
▸ 基于唤醒词前缀的指令系统，支持 @某人 方式指定目标
▸ 记录查询支持汇总统计与图片表格详情（含彩色状态标注）
▸ 处罚操作前检查成员在群状态，避免无效操作
▸ 撤销禁言时自动解禁，撤销踢黑时自动移出黑名单
