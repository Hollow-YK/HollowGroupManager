# 多群联动管理Bot

基于 [QFun](https://github.com/YuShengXiang1/QFun-Plugin) 框架的 QQ 群违规管理插件。支持将多个 QQ 群组成管理组，实现跨群同步处罚、记录共享与黑名单联动。

## 功能概览

- **多群联动** — 一个管理群 + 多个执行群，组内处罚自动同步
- **处罚操作** — 踢人 / 禁言 / 警告，支持拉黑（kick f）
- **三步检查** — 所有操作前验证"成员在群 → 状态检查 → 执行"
- **撤销机制** — 撤销禁言自动解禁，撤销踢黑自动移出黑名单
- **记录查询** — 汇总统计 + 图片表格详情（彩色状态标注）
- **黑名单** — 组黑名单成员入群自动踢出
- **权限分级** — 超级管理员 / 管理员 / 普通成员三级

## 快速开始

1. 从 [Releases](https://github.com/Hollow-YK/HollowGroupManager/releases) 下载最新版 `HollowGroupManager_QF-x.x.x.zip`

2. 解压得到 `多群管理Bot/` 文件夹，放入 QFun 插件目录：

   ```text
   /storage/emulated/0/Android/data/com.tencent.mobileqq/QFun/<QQ号>/plugin/
   ```

3. 编辑 `多群管理Bot/config.properties`：

   ```properties
   wakeWords=/,!,。
   superAdmins=你的QQ号,另一个超级管理员QQ
   dataDir=data
   ```

4. 在 QFun 中加载插件，在目标群发送 `/group admin 管理组名称` 创建管理组

5. 在其他需要联动的群发送 `/group set 管理组名称` 加入执行群

6. 发送 `/help` 查看可用指令

## 指令列表

> `[唤醒词]` 代表 config.properties 中配置的唤醒前缀（默认 `/`、`!`、`。`）。例如唤醒词为 `/` 时，help 指令即为 `/help`。
>
> `QQ用户` 支持通过 @某人 或直接输入QQ号的方式指定。

<details>
<summary><b>/p — 处罚成员</b></summary>

**权限**：0 / 1

**格式**：`[唤醒词]p <QQ用户> <方式> [内容] <原因>`

**参数说明**：

| 参数 | 必填 | 说明 |
| ---- | ---- | ---- |
| `<QQ用户>` | 是 | 被处罚者。支持 `@某人` 或直接输入 QQ 号 |
| `<方式>` | 是 | 处罚类型：`kick` 踢出 / `mute` 禁言 / `warning` 警告 |
| `[内容]` | 视方式而定 | `kick`：可选 `f`（加入黑名单）；`mute`：必填时长；`warning`：不需要 |
| `<原因>` | 是 | 处罚原因，用于记录和通报。缺少原因时不会执行，仅记录为"不合规" |

**禁言时长格式**：

| 格式 | 示例 | 含义 |
| ---- | ---- | ---- |
| 纯数字 | `1` / `0.5` | 1 天 / 0.5 天（12 小时） |
| 组合格式 | `1d2h30m` | 1 天 2 小时 30 分钟 |
| 组合格式 | `3h` | 3 小时 |
| 组合格式 | `30m` | 30 分钟 |

**执行逻辑**：遍历所有执行群，对每个群执行三步检查：

1. 成员在群？（不在则静默跳过，不视为失败）
2. 状态检查（禁言时若已被禁言则更新时长）
3. 执行操作 → 群内通报 `@某人 因「原因」被XX`

**示例**：

```text
/p @某人 mute 1d2h 广告刷屏     → 禁言 1 天 2 小时
/p 123456 kick f 严重违规        → 踢出并加入黑名单
/p @某人 warning 注意言辞        → 警告（仅记录通报）
/p @某人 kick 长期潜水            → 踢出但不加入黑名单
/p 123456 mute 3h 刷屏            → 禁言 3 小时
```

</details>

<details>
<summary><b>/h — 查询记录</b></summary>

**权限**：0 / 1

**格式**：`[唤醒词]h [QQ用户] [-i]`

**参数说明**：

| 参数 | 必填 | 说明 |
| ---- | ---- | ---- |
| `[QQ用户]` | 否 | 要查询的成员。**不填则显示当前管理组全部处罚记录** |
| `[-i]` | 否 | 详情模式开关。不带 `-i` 输出汇总统计；带 `-i` 输出图片表格详情 |

**三种模式对比**：

| 模式 | 输出形式 | 包含内容 |
| ---- | ---- | ---- |
| 无参数 | 图片表格 | 管理组内全部处罚记录 |
| 汇总（指定成员，不带 `-i`） | 文字 | 被处罚总次数、禁言次数/总时长、被踢次数 |
| 详情（指定成员，带 `-i`） | 图片表格 | ID、时间、发起群、发起者、方式、内容、原因、状态、撤销时间、撤销原因 |

状态颜色：🟢已执行 / 🟠已撤销 / 🔴执行失败 / 🟤部分失败 / ⚫不合规

**示例**：

```text
/h               → 显示管理组全部记录
/h @某人          → 汇总统计
/h 123456 -i     → 图片表格详情
```

</details>

<details>
<summary><b>/rp — 撤销处罚</b></summary>

**权限**：0 / 1

**格式**：`[唤醒词]rp <记录ID> [撤销原因]`

**参数说明**：

| 参数 | 必填 | 说明 |
| ---- | ---- | ---- |
| `<记录ID>` | 是 | 要撤销的处罚记录 ID（数字）。可通过 `/h <QQ用户> -i` 查询获取 |
| `[撤销原因]` | 否 | 撤销原因，可包含空格。不填则仅记录撤销操作 |

**限制**：

- 仅可撤销状态为"已执行"、"执行失败"或"部分失败"的记录
- 不合规、已撤销的记录不可再次撤销
- 记录 ID 不存在时报错

**撤销禁言时**，对每个执行群执行三步检查：

1. 成员在群？
2. 处于禁言状态？
3. 执行解禁 `shutUp(0)`

**撤销踢黑时**，自动移出管理组黑名单。撤销 kick（不带 f）或 warn 无实际操作，仅更新记录状态。

**示例**：

```text
/rp 5                  → 撤销记录 5，不填原因
/rp 5 误判，实际未违规   → 撤销记录 5，附撤销原因
```

</details>

<details>
<summary><b>/a — 设置权限</b></summary>

**权限**：0（仅超级管理员）

**格式**：`[唤醒词]a <QQ用户> [1/-1]`

**参数说明**：

| 参数 | 必填 | 说明 |
| ---- | ---- | ---- |
| `<QQ用户>` | 是 | 要设置权限的成员。 |
| `[1/-1]` | 否 | 目标权限等级。`1` = 管理员，`-1` = 普通成员。不填默认为 `1` |

**约束**：

- 不可设为 `0`（超级管理员只能在 `config.properties` 的 `superAdmins` 中配置）
- 不可修改自己的权限等级
- 非法等级值（如 `0`、`2`、`abc`）会提示错误

**示例**：

```text
/a @某人               → 设为管理员（默认）
/a 123456 1            → 设为管理员
/a 123456 -1           → 降为普通成员
```

</details>

<details>
<summary><b>/group — 管理组配置</b></summary>

**权限**：0（仅超级管理员）

**子命令**：

| 子命令 | 格式 | 说明 |
| ------ | ---- | ---- |
| `admin` | `[唤醒词]group admin <组名>` | 将当前群设为**管理群**并创建管理组 |
| `set` | `[唤醒词]group set <组名>` | 将当前群加入管理组作为**执行群** |
| `remove` | `[唤醒词]group remove` | 将当前群移出管理组 |
| `info` | `[唤醒词]group info` | 查看当前群所在管理组信息 |

**各子命令参数**：

**`admin`** — `<组名>`：管理组名称（必填）。不可与已有管理组重名，当前群不可已属于其他管理组。

**`set`** — `<组名>`：目标管理组名称（必填）。管理组必须已存在，管理群本身不能作为执行群，当前群不可已属于其他管理组。

**`remove`** — 无参数。管理群无法通过此命令移出（需重建管理组）。

**`info`** — 无参数。显示组名、当前群角色（管理群/执行群）、管理群号、执行群列表。

**示例**：

```text
/group admin 稽核组      创建管理组「稽核组」，当前群为管理群
/group set 稽核组        将当前群加入「稽核组」作为执行群
/group info              查看当前群所在管理组信息
/group remove             从管理组移出
```

</details>

## 权限等级

| 等级 | 角色 | 权限范围 |
| --- | --- | --- |
| 0 | 超级管理员 | 全部指令，不可被修改等级 |
| 1 | 管理员 | 处罚、查询、撤销 |
| -1 | 普通成员 | 不响应任何指令 |

超级管理员在 `config.properties` 的 `superAdmins` 中配置；管理员通过 `/a` 指令设置。

## 管理组架构

```text
管理群（adminGroup）
  └── 执行群A ── 同步执行处罚
  └── 执行群B ── 同步执行处罚
  └── 执行群C ── 同步执行处罚
```

- 在**管理群**中发起的处罚会自动同步到所有**执行群**
- 成员不在某执行群时静默跳过，不视为失败
- 黑名单跨所有执行群共享

## 操作检查流程

### 处罚（kick / mute / warning）

```text
① getGroupMemberList → 成员在群？
② getProhibitList（仅 mute）→ 已禁言？通知管理群
③ 执行操作
④ 执行后验证 → kick：检查成员是否仍在群内  mute：检查成员是否在禁言列表
⑤ 验证通过 → 群内通报「@某人 因「原因」被XX」
  验证失败 → 标记执行失败/部分失败，通知管理群
```

### 撤销禁言

```text
① getGroupMemberList → 成员在群？
② getProhibitList → 处于禁言状态？
③ shutUp(0) → 解除禁言
```
