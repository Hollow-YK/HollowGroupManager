多群联动管理Bot — 版本历史
============================

V1.0.4 (2026-06-24)
--------------------
▸ 修复执行失败被当作成功的 bug：多群执行时部分群失败不再错误标记为"全部失败"，正确区分"执行失败"与"部分失败"
▸ 新增"部分失败"记录状态：部分执行群失败时标记为"部分失败"，表格渲染使用暗红色（#B71C1C）标注，撤销操作同时支持"部分失败"记录
▸ 处罚操作增加执行后验证：踢人后遍历群成员列表确认目标已不在群内、禁言后检查禁言列表确认目标已被禁言，bot 权限不足时立即标记失败
▸ 黑名单入群踢出也增加执行后验证，踢出失败时写入 error.log
▸ 修复 help 指令回复样式问题
▸ 管理群通报消息增加执行状态与失败详情（状态 + 失败群列表），便于追踪每笔处罚结果
▸ 更新 CI release 流程与项目文档，新增版本详细对比文档和开发指南

V1.0.3 (2026-06-18)
--------------------
▸ help 指令支持参数化查询（/help、/help p、/help h 等），返回对应指令的详细帮助
▸ help 指令改为图片表格方式回复，排版清晰，含权限标注与使用示例
▸ h 指令新增无参数模式，直接输出管理组内全部处罚记录的图片汇总表格
▸ 全局可变状态（groups / records / permissions 等）增加并发安全保护，防止多线程数据竞争
▸ 新增 recordsById 索引，加速按记录 ID 查找的查询与撤销操作
▸ JSON 解析器重构：精简代码、增强容错，对非法 JSON 数据做防御处理
▸ 所有指令的 QQ 用户参数位置统一支持 @某人 方式指定目标
▸ 启动时对 config.properties 做完整性验证，缺少必填字段或非法值时给出明确提示
▸ 更新 CI release 流程配置，优化发版自动化

V1.0.2 (2026-06-17)
--------------------
▸ 所有处罚操作前增加三步检查（成员在群 → 状态检查 → 执行），不在群则静默跳过
▸ 解除禁言前验证成员在群且处于被禁言状态，避免无效操作
▸ 成员身份检查改用 getGroupMemberList 遍历比对，比 getMemberInfo 更可靠
▸ 处罚成功后在各执行群内通报「@某人 因「原因」被处罚」
▸ 详细记录查询（-i）改为图片表格展示，含彩色状态标注与交替行底色
▸ 修复 BeanShell 环境下 ((Number) cast) 导致的 bsh.Primitive 类型转换异常
▸ 修复与 Java 内置 java.lang.Record 的类名冲突（重命名为 PunishRecord）
▸ 完善空值保护，防止旧数据 revokeReason/content 字段导致的空指针异常

V1.0.1 (2026-06-17)
--------------------
▸ 修复会对不存在的成员进行处罚的 bug
▸ 新增 gitignore

V1.0 (2026-06-17)
--------------------
▸ 初始发布
▸ 支持多群联动管理组（管理群 + 执行群）
▸ 支持踢人、禁言、警告、撤销、记录查询
▸ 黑名单自动拦截入群
▸ 基于唤醒词前缀的指令系统
▸ config.properties 配置超级管理员与唤醒词
